Операционные системы и среды

Краткое описание: это основная программа компьютера, которая управляет всеми его процессами. Она запускает программы, хранит файлы и контролирует работу устройств.

Описание

Что такое ОС?

Операционная система — это невидимый «дирижёр» компьютера, который управляет всеми процессами и заставляет всё работать слаженно. Она распределяет память, контролирует устройства и запускает программы, как будто руководит целой командой. Именно благодаря ей ты можешь открывать приложения, играть, учиться и пользоваться интернетом. Без операционной системы компьютер был бы просто набором «железа», который ничего не умеет. Примеры: Windows, Linux.

Роль ОС

ОС превращает сложное «железо» компьютера в удобный и понятный инструмент для программ и человека. Она создаёт виртуальный мир, где программы могут работать, не думая о деталях процессора или памяти. Благодаря ОС каждая программа получает то, что ей нужно, а пользователь — плавную и безопасную работу. Проще говоря, ОС — это мост между человеком и машиной, который делает работу с компьютером простой и удобной.

Среды (оболочки):

Графические и командные оболочки превращают компьютер в удобное рабочее пространство. Графическая среда показывает иконки, окна и меню, позволяя работать мышкой и видеть всё на экране. Командная среда даёт полный контроль через текстовые команды для опытных пользователей. Проще говоря, оболочка — это «окно» в мир компьютера, через которое мы общаемся с техникой удобно и эффективно.

Лаборатория

Здесь можно добавить свои лабораторные задания и описание.

Лабораторная 1: Графический интерфейс пользователя

Цель: Изучить основы GUI и разработать простое приложение с использованием Tkinter или аналогичной библиотеки.

Пример: Калькулятор с кнопками 0-9, операциями +, -, *, / и текстовым полем для результата.

Задание:

  • Создать окно с заголовком "Моя первая программа GUI".
  • Добавить текстовое поле для ввода.
  • Кнопки: "Очистить", "Вычислить", "Выход".
  • Реализовать суммирование двух чисел при нажатии "Вычислить".
  • Очистка полей и завершение приложения.

Итог: Освоены базовые элементы GUI (окна, кнопки, метки), изучены принципы компоновки (grid). Приложение корректно выполняет вычисления и управляет вводом.

Вывод: GUI упрощает взаимодействие с пользователем, получены навыки для более сложных приложений.

Приложения: Скриншоты приложения, исходный код.

Лабораторная 2: Архитектуры операционных систем

Цель: Изучить монолитные, микроядерные и гибридные архитектуры ОС, сравнить Linux и Windows.

Пример: Монолитная (Linux) — высокая производительность; Микроядро (Minix) — надежность; Гибридная (Windows NT) — баланс.

Задание:

  • Изучить теорию архитектур ОС (монолитная, микроядро, гибридная).
  • Анализ ядра Linux (uname, /proc/version, /proc/cpuinfo).
  • Сравнить с Windows (msinfo32, Диспетчер задач).
  • Сделать вывод о применимости архитектур.

Итог: Понимание особенностей каждой архитектуры и их применения в разных задачах (серверы/десктопы).

Вывод: Современные ОС используют гибридные решения для баланса производительности и надежности.

Приложения: Скриншоты команд, таблица сравнения архитектур.

Лабораторная 3: Механизм обработки прерываний

Цель: Изучить работу прерываний, распределение IRQ, нагрузку на систему.

Пример: Анализ /proc/interrupts для CPU0/CPU1, наблюдение за сетевыми и дисковыми IRQ.

Задание:

  • Анализ текущих прерываний Linux.
  • Искусственные нагрузки: USB, сеть, диск.
  • Анализ Windows через Диспетчер устройств.

Итог: Изучен механизм обработки прерываний, балансировка нагрузки на CPU, влияние на производительность.

Вывод: Понимание IRQ важно для оптимизации системы и настройки драйверов.

Приложения: Скриншоты /proc/interrupts, графики нагрузки.

Лабораторная 4: Управление процессами в Windows

Цель: Освоить инструменты управления процессами и потоками в Windows, приоритетами и планировщиком.

Пример: PowerShell: Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

Задание:

  • Мониторинг процессов через Диспетчер задач.
  • CLI: tasklist, taskkill для управления процессами.
  • PowerShell: анализ топ-10 по CPU и памяти.
  • Запуск процессов с разными приоритетами.
  • Эксперимент: наблюдение нагрузки на CPU и планировщик.

Итог: Освоены мониторинг, управление процессами, приоритизация, PowerShell для детального анализа.

Вывод: Эффективное управление процессами позволяет оптимизировать производительность и предотвращать зависания.

Приложения: Скриншоты Диспетчера задач, вывод PowerShell, таблица приоритетов.

Лабораторная 5: Управление памятью в ОС

Цель: Изучить виртуальную память, свопинг, регистры база-предел и мониторинг памяти в Windows.

Пример: Task Manager → Производительность → Память: Использовано 12 ГБ / 32 ГБ.

Задание:

  • Мониторинг памяти через Task Manager и Resource Monitor.
  • Эмуляция нагрузки: mspaint, браузер, видео.
  • Эмуляция регистров база-предел, вычисление физических адресов.
  • Анализ свопа и страниц памяти.

Итог: Освоено абстрагирование памяти, своп, таблицы страниц и мониторинг нагрузки.

Вывод: Виртуальная память обеспечивает изоляцию процессов и эффективное использование ресурсов.

Приложения: Скриншоты Task Manager, Resource Monitor, расчеты база-предел.

Лабораторная 6: Эволюция управления памятью. Мультипрограммирование

Цель: Изучить эволюцию управления памятью и принципы мультипрограммирования.

Пример: Фиксированные зоны → переменное разбиение → страницы/сегменты → виртуальная память + своп.

Задание:

  • Теоретический анализ эволюции управления памятью.
  • Эксперимент мультипрограммирования: запуск 1, 5, 20 процессов и измерение нагрузки.
  • Эмуляция внутренней и внешней фрагментации.
  • Определение коэффициента загрузки памяти и оптимальной степени мультипрограммирования.

Итог: Понимание исторического развития памяти, фрагментации, виртуальной памяти и мультипрограммирования.

Вывод: Современная виртуальная память обеспечивает изоляцию процессов и эффективное распределение ресурсов.

Приложения: График "Процессы vs Память/CPU", скриншоты Task Manager.

Лабораторная 7: Управление вводом/выводом в ОС Windows

Цель: Изучить принципы работы I/O подсистемы, буферизацию, кэширование, асинхронный ввод/вывод.

Пример: Resource Monitor → Диск: чтение/запись процессов.

Задание:

  • Мониторинг I/O через Resource Monitor и Диспетчер задач.
  • Performance Monitor: Disk Time, Disk Bytes/sec, Avg. Disk Queue Length.
  • Тестирование скорости чтения/записи через PowerShell.
  • Управление устройствами и настройка кэширования.
  • Асинхронные операции I/O (PowerShell jobs).

Итог: Изучены методы мониторинга и управления I/O, выявлены узкие места.

Вывод: Подсистема I/O использует многоуровневую буферизацию, кэширование и асинхронные операции для минимизации задержек.

Приложения: Resource Monitor, Perfmon графики, тесты скорости чтения/записи.

Лабораторная 8: Системы управления файлами в ОС Windows

Цель: Изучить файловые системы NTFS, FAT32, exFAT, структуру, права доступа и журналирование.

Пример: fsutil fsinfo volumeinfo C: → размер кластера, свободное место, журнал.

Задание:

  • Анализ файловой системы и MFT.
  • Сравнение файловых систем по кластеру, журналу, ACL, максимальному файлу.
  • Управление кластерами и правами доступа (fsutil, icacls).
  • Тест фрагментации (defrag).

Итог: Изучены NTFS: кластеры, MFT, журналирование, права доступа, оптимизация фрагментации.

Вывод: NTFS — современная файловая система с безопасностью, журналированием и эффективным управлением пространством.

Приложения: fsutil fsinfo, таблица прав icacls, отчет defrag.

Лабораторная 9: Работа с интерфейсом командной строки Windows (CMD)

Цель: Освоить команды CMD, пакетные файлы, переменные окружения, автоматизацию задач.

Пример: dir *.txt /b /s > filelist.txt; for %i in (*.txt) do echo %i

Задание:

  • Навигация и управление файлами (cd, dir, copy, xcopy, robocopy).
  • Перенаправление потоков (>, >>, |).
  • Использование переменных и условий (set, if).
  • Циклы и пакетные файлы (.bat) для автоматизации.
  • Сетевые команды и системное управление (ping, tracert, netstat, ipconfig, tasklist, taskkill, sfc, chkdsk, wmic).

Итог: Освоены базовые и продвинутые возможности CMD, создание скриптов для автоматизации.

Вывод: CMD — мощный инструмент для системного администрирования и рутинной автоматизации без GUI.

Приложения: Созданные .bat файлы, скриншоты работы скриптов, вывод system.txt.
